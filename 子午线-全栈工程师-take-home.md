## 背景
子午线（Meridian）做 **AI + B2B 出海销售**：用 AI Agent 帮助企业自动化地完成海外 B2B 获客与销售。本作业是一个贴近我们业务的小型「全栈 + Agent」任务。

## 任务：AI 销售线索助手（Mini Lead Assistant）
做一个小应用：用户输入一个目标公司的**名称 / 官网 / 一句话描述**，后端用一个**带工具调用的 LLM Agent** 产出结构化的「销售线索简报」，前端展示出来。

### 后端（Python，建议 FastAPI）
- 提供 `POST /api/lead`，入参为公司名 / 网址 / 简述。
- 实现一个**最小 Agent 循环**：至少包含 **1 个可被调用的工具**（例如 `search_company(query)`，用 mock 假数据或真实网络请求都可以），由 LLM 决定是否调用工具并整合结果。
- 输出**结构化 JSON**：公司概况、可能的痛点、推荐的出海 / 外联切入点、一句话开场白（outreach opener）。
- 返回中附上 Agent 的**工具调用 / 思考轨迹**，方便我们看你的 Agent 设计。
- LLM 可用任意 API（Claude / OpenAI / 通义等）或本地 / stub；密钥用**环境变量**读取，并在 README 写明。

### 前端（React）
- 输入框 + 提交按钮 + 结果展示区，呈现结构化简报与轨迹。
- 处理 **loading / 错误** 状态。
- 样式从简，清晰即可。

### 约束（务必控制在 4 小时内）
- 无需登录、无需数据库（内存即可）。
- 工具与数据可以 mock；重点是**全栈打通 + Agent 工具调用的设计**，不是数据真假。
- 鼓励使用 AI 编码工具（我们就是 Vibe Coding 的拥趸），但请确保你理解每一行代码。
